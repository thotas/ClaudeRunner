Claude Runner

Drop a text file, get autonomous code changes — three-phase Claude Code pipeline with guardrails
Autonomous 3-Layer Guardrails Cron-Ready Zero Dependencies
Pipeline
1
pick_task()
Selects the oldest .txt from tasks/pending/ (FIFO queue)
2
phase_review()
Claude identifies project, classifies task type, enriches with actionable detail
3
merge_guardrails()
Combines global + per-project + per-task guardrails into a single constraint set
4
phase_execute()
Runs Claude Code in the project directory — branch, code, test, commit, push
5
phase_report()
Generates a structured markdown report from the execution log
6
notify()
Calls pluggable hook — WhatsApp, Telegram, Slack, or just a log
Key Features
  • Free-text tasks — no structured format required
  • Auto-detects project from aliases in registry
  • Code tasks: branch, implement, test, commit, push
  • Non-code tasks: code review, security audit, research reports
  • Three-layer guardrails prevent unsafe operations
  • Atomic lock prevents concurrent runs
  • SIGINT/SIGTERM traps restore tasks to queue
  • Dry-run mode previews Phase 1 without executing
Stack
Bash 3.2+ Claude Code CLI jq MiniMax API cron git
